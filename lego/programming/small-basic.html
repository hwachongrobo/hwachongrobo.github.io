Small Basic

EV3 Basic is an alternative method of programming for EV3 Mindstorms. It uses Microsoft Small Basic IDE and its corresponding EV3 extension

For more information visit, Microsoft Small Basic or EV3 Basic Extension

Why use Small Basic EV3?#

You might already be familiar with EV3-G and EV3 Classroom.

  • EV3-G
  • EV3 Classroom
  • Small Basic EV3

for i = 1 to 4    Motor.Move("BC", 50, 720, "True")    Motor.MoveSync("BC", 30, 0, 320, "True")EndFor

However, they both come with their sets of issues. For example, EV3-G is known to be very laggy, particularly with large codebases;

EV3 Classroom and Small Basic EV3 does not support the use of HiTechnic Color Sensors.

Small Basic doesn't require you to load up an SD card and wait spend a long time ⌛ just to wait for a firmware update.

At the same time , you can also you full fludged programming techniques such as arrays that is not support in EV3-G or EV3 Classroom.

Getting Started#

Before you get your hand dirty and start programming like the Russians, you might want to read up on the syntax of Microsoft Small Basic as it is a proprietary language by Microsoft.

Once you are done with that, go download the Small Basic IDE and the EV3 Basic Extension

Take note that it only works for Windows, those using macOS can just click off of this guide and miss out on the benefits of using small basic 😄. That is what you get for paying for an overly priced laptop 😈

So how are you supposed to download the EV3 Basic code and run on your EV3 🤔

Go to your Windows start menu by clicking on the little Windows icon and find for EV3 Explorer. The left side of the screen is your EV3 and the right side is your computer. Find for your code on the right, and create/navigate to a folder you made in the EV3 brick on the left. Once you are in your desired path, click Download on the bottom right of the screen. Then click on the file on the left side and click on Compile and you should be presented with an executable.

Basic Movment#

Motor.Invert("Port number")

Like the name suggests, it inverts the motor speed automatically when the code is compiled.

Motor.GetCount(port)

This is to get the number of degrees the motor has made

Motor.MoveSync(port,speed for port 1, speed for port 2, degrees,brake)

This is what I would use for basic movement as you can control the speed of each motor individually. Its the excat replica of the Move Tank block in EV3-G

Sensors#

Light Sensor

Sensor.SetMode(port,mode number)

For EV3 Sensors, there are usually 3 Modes,

1 - Color
2 - Reflected Light Intensity
3 - Ambient Light Intensity

However, for Small Basic, it opens up another mode there is more powerful and more accurate than the other three. Its called the Raw RGB mode. It is the fourth mode in the EV3 Basic syntax and it reads the raw RGB value of the light sensor - Red, Green and Blue. Ranging from 0 - 500 🤯. Therefore, it doesn't matter if your sensor position, whether it is too high or too close to the ground. So once you have set your mode number to 4 and want to unlock the UNLIMITED POWERRR of that color sensor mode. How are you supposed to read it then?
Well, if you have been reading the guide throughly (I know some of you do not 😏) you use the TextWindow.WriteLine("") and Sensor.ReadRaw() classes.

lightIntesntiy = Sensor.ReadRaw(port,3)TextWindow.WriteLine(lightIntenstiy)

❗❗❗ Remember you cannot compile it in EV3 Explorer just run it directly from the IDE ❗❗❗

What would you see is a pop up menu that shows you three arrays, 0=x, 1=x, 2=x, (arrays start at zero). Using common sense, you would add up all of the values and💥that is your reflected light value.
